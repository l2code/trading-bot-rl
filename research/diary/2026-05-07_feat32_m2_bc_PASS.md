## Why this diary exists

M1 (PR #86) shipped the v3 chronological env scaffold but proved nothing about whether the env is *learnable* — only that the plumbing works end-to-end. M2's question, scoped per the FEAT-32 plan: **can a supervised classifier imitate a non-trivial state-dependent target policy on this env's obs?** If not, we have an env problem that no PPO retrain will fix and the right move is to back up. If yes, the obs is sufficient to recover a state-dependent decision rule and we earn the right to run M3 (Kaggle masked-PPO 500k×3).

The diagnostic mirrors the FEAT-30 supervised-ranker pattern from v2: a cheap classifier acts as the "is RL learning anything?" canary. v2's lesson was that masked-PPO collapsed to bit-identical first_fired behavior on this benchmark; v3's M2 establishes a learnability floor before we spend Kaggle compute.

## What we trained

A hand-coded `BCTargetPortfolioPolicy` with three deterministic action regions:

```
action 2  if n_actions ≥ 3 AND n_fired ≥ 2 AND signal_gap_top2 < 0.10
action 1  if n_fired ≥ 1 AND cash_pct > 0.50
action 0  otherwise
```

The target reads three obs slots — `n_fired` (idx 0), `signal_gap_top2` (idx 4), and `cash_pct` (idx 6) — out of the 12-dim obs. By design it has action variance across states (constant-label targets like "always top1" produce trivial 100% accuracy and prove nothing). The supervised model is `sklearn.ensemble.HistGradientBoostingClassifier(max_iter=200, max_depth=8, random_state=11)`.

Train: 80 episodes × 60-day random windows over `train_start=2014-01-01..train_end=2020-12-31` (yfinance_daily, starter_equities universe) → 4,800 (obs, action) pairs.
Val: 20 episodes × 60-day random windows on a different seed → 1,200 (obs, action) pairs.

## Verdict reasoning

The classifier recovers the deterministic 3-region target perfectly on a held-out window split. Three readings:

1. **The 12-dim obs carries enough signal to express a non-trivial state-dependent policy.** The target uses only 3 of the 12 obs dims; HistGB also has access to the other 9 and could in principle confuse itself, but doesn't. The obs space isn't degenerate.
2. **Train and val windows draw from the same 7-year regime** (random 60-day windows from 2014-2020), so 100% accuracy is a learnability floor, not a generalization claim. M2's question is "can we even fit?" — the answer is yes. A regime-shift generalization test would belong to M4.
3. **The class distribution is non-degenerate.** Action 2 dominates (56%), action 1 is rare (8%), action 0 is common (36%). The classifier didn't collapse to "predict the majority class" — every cell of the diagonal is filled with the right count.

This rules out one of the two possible failure modes for M3:
- ✅ M2 PASS: env is learnable in principle (this diary).
- ⏳ M3 will tell us: can PPO discover a profitable policy by RL, given that supervised methods can fit one by imitation?

The reason this distinction matters: in v2, masked-PPO collapsed to first_fired and supervised ranker also failed the gate. There, both methods failed together, so neither one alone diagnosed the problem. In v3, supervised passes; if PPO collapses to top1 or no_op (the M1 smoke pattern), that's *PPO* failing, not the *env* failing. M2 disentangles the two.

## What this does NOT prove

- **Not a profitability claim.** The target policy is hand-coded, not optimized for return. We only confirmed that an obs-conditional rule is *learnable*. M3 has to find a *profitable* obs-conditional rule.
- **Not a regime-stability claim.** Train and val both drew from 2014-2020. The 2022 yfinance test (M4) is held out and untouched.
- **Not a comparison vs `portfolio_baseline_top1`.** The M2 plan asked for "comparison vs portfolio_baseline_top1." That comparison happens via `rl-swing validate` once the BC artifact is on disk; the variant auto-includes `portfolio_baseline_bc` in evaluate(). The val-acc number above is the env-learnability check the plan named as the primary acceptance criterion. The validate-time comparison is informational not gating, since the BC target was hand-coded for diagnostic variance not for return.

## Decision point — proceed to M3?

**Yes.** M2 PASS satisfies the env-learnability gate. M3 scope (per the FEAT-32 plan):

- Kaggle private masked-PPO retrain on v3 chronological env, 3 seeds × 500k steps.
- Action lattice `Discrete(1 + max_top_k)` with no slot-strategy mask (the action lattice is small enough that masking is redundant for now; revisit at M4 if seeds collapse).
- Acceptance gate: `validation_composite_score` strictly greater than `portfolio_baseline_random` (Phase-24 gate equivalent), per cycle, per seed.
- ~45 min Kaggle wall-time at default ent_coef=0.01; budget overrun escalation per CLAUDE.md §3.3 default-OFF rule.

If M3 PASSES gate (≥2 of 5 metrics improved + no material regression on the strongest baseline): proceed to M4 multi-cycle yfinance per D4-b harness (2021/2022/2023/2024).

If M3 FAILS (PPO collapses to no_op, top1, top2, or random_action equivalent — the M1 smoke pattern): the right move is the FEAT-32 plan's M3.b fallback (Optuna sweep on ent_coef + lr). M2 already eliminated the env-degeneracy hypothesis, so an M3 collapse would be a hyperparam / exploration problem, not an architecture problem.

## Honest annotations

- **Why 1.0000 is suspicious-looking but defensible:** the target is deterministic. A perfect classifier on a deterministic target with iid splits is the expected result, not a red flag. The flag would be ≤70% — which would mean the obs *can't* express the rule at all. We're far above the floor.
- **Per-class distribution skew (56% action 2):** comes from the rule + slate distribution, not from a degenerate target. Most slates have ≥2 fired packs with similar signal_strength (because momentum + breakout often co-fire on the same day), so action 2 dominates. action 1 is comparatively rare because it requires (n_fired ≥ 1) AND (cash > 0.5) AND NOT (n_fired ≥ 2 AND gap < 0.10) — a narrow band.
- **What would have failed M2:** signal_gap_top2 not being computed correctly in the obs, or n_fired being clipped/normalized in a way that lost the {0, 1, 2+} distinction, or cash_pct not updating across days. Any of those would have given a classifier ≤70% accuracy. The fact that we hit 100% means each of these obs slots faithfully reflects the underlying state.
